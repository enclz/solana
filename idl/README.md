# Generated IDL artifacts

This directory holds the committed Anchor IDL and derived error map for the
Enclz program. Contents are regenerated by `npm run idl:sync` (which delegates
to `scripts/sync-idl.mjs`) after `anchor build`, and are kept in sync with
`target/idl/enclz.json` by the `idl-drift` job in `.github/workflows/program-ci.yml`.

## Files

- `enclz.json` — the program's Anchor IDL. Regenerated from
  `target/idl/enclz.json`, committed so the backend / external integrators can
  consume it without running the Anchor toolchain. **Will be populated after
  the first successful devnet deploy** (`npm run deploy:devnet`).
- `error-map.json` — extracted from the IDL's `errors[]`. Each entry maps an
  Anchor error code (offset 6000) and variant name to the camelCased REST error
  code the backend exposes. Stable variant names guarantee the backend stays
  in sync with the program; CI fails on any drift.

## Updating

```bash
anchor build
npm run idl:sync
git add idl/enclz.json idl/error-map.json
```

CI will reject any PR that modifies the program's public surface without a
matching update here.
